# Bug Fix Plan: Category Filter Only Works on First Click

## Root Cause Analysis

### The Bug
When a user clicks a category chip, products display correctly the first time. On subsequent clicks (different category or "All"), the product grid appears empty even though the data is correct.

### Root Cause: `fade-in` CSS + One-Shot IntersectionObserver

The bug is caused by the interaction between two things:

1. **CSS** (`css/main.css` lines 529-538): Every product card has class `fade-in`, which starts at `opacity: 0; transform: translateY(30px)`. The card only becomes visible when the `.visible` class is added.

2. **JS** (`js/app.js` lines 179-195): `observeFadeIns()` creates an IntersectionObserver **once** during `init()` and observes all `.fade-in` elements present in the DOM at that time. When those elements intersect the viewport, `.visible` is added.

**The failure chain:**
1. Page loads → all 20 products rendered → `observeFadeIns()` observes them → they get `.visible` → visible to user
2. User clicks "category B" → Alpine.js `x-for` destroys non-matching cards, keeps matching ones (which already have `.visible`) → products appear correctly (first click works)
3. User clicks "category C" → Alpine.js destroys category B cards, creates **new** DOM elements for category C products → these new elements have `class="menu-card fade-in"` (opacity: 0) but are **never observed** by the IntersectionObserver (it was set up once in step 1 and doesn't know about these new elements) → cards remain invisible → grid appears empty

### What is NOT the cause
- No event listener issues (Alpine.js `@click` bindings are declarative and always work)
- No caching problems (localStorage caching is only for initial load fallback)
- No pagination/offset bugs (there is no pagination - all products load at once)
- No SQL query issues (filtering is entirely client-side via the `filteredProducts` getter)
- No state mutation bugs (the getter is pure and reactive)
- No race conditions (data loads once via `await` in `init()`)

## Fix

### Single file to modify: `js/app.js`

Three changes in the Alpine.js component:

**Change 1**: Store the IntersectionObserver as a reusable component property (add `_observer: null` to state)

**Change 2**: Update `observeFadeIns()` to:
- Reuse the existing observer if already created (don't create a new one each time)
- Only observe `.fade-in` elements that don't already have `.visible` (avoid re-observing already-visible elements)

**Change 3**: Update `setCategory()` to call `observeFadeIns()` via `this.$nextTick()` after setting `activeCategory`, so the new DOM elements get observed after Alpine finishes re-rendering

### Exact changes

```js
// Line ~18: Add observer property
_observer: null,

// Lines 103-105: Update setCategory to re-observe after render
setCategory(catId) {
  this.activeCategory = catId;
  this.$nextTick(() => this.observeFadeIns());
},

// Lines 179-195: Update observeFadeIns to reuse observer
observeFadeIns() {
  if (!this._observer) {
    this._observer = new IntersectionObserver(
      (entries) => {
        entries.forEach((entry) => {
          if (entry.isIntersecting) {
            entry.target.classList.add("visible");
            this._observer.unobserve(entry.target);
          }
        });
      },
      { threshold: 0.1, rootMargin: "0px 0px -40px 0px" }
    );
  }
  document.querySelectorAll(".fade-in:not(.visible)").forEach((el) => {
    this._observer.observe(el);
  });
},
```

### Why this fix is safe
- Only touches JavaScript logic, no HTML/CSS changes
- Preserves the fade-in animation (new cards still animate in)
- The observer is reused efficiently (not recreated each time)
- `:not(.visible)` prevents double-observing already-visible elements
- `$nextTick` ensures Alpine has finished DOM updates before we observe

## Verification Plan

After applying the fix, test manually by opening `index.html` in a browser:

1. **Initial load**: All products should display (all categories)
2. **Click specific category** (e.g., "قهوه‌های گرم"): Only that category's products should appear
3. **Click different category** (e.g., "نوشیدنی‌های سرد"): Products should appear (this is where it previously broke)
4. **Click "همه" (All)**: All products should reappear
5. **Rapid switching**: Click through all categories quickly - products should always load
6. **Category with zero products**: Should show empty state correctly
7. **Search + category combo**: Type in search, switch categories, verify results
8. **Check fade-in animation**: New products should still animate in smoothly (not just pop in)
