# Fix Plan — Footer, Search, About Logo, Credit, Dark Mode

## Changes

### 1. Footer Background (`css/main.css`)
**Problem**: Footer uses `--cafe-darker` (#2a2018) — too dark, logo/text not clearly visible.
**Fix**: Change `.site-footer` background to warm cream/nescafe color (`#e8ddd0` or `#d4c4b0`), update text colors to dark brown for contrast.

- Change `--cafe-darker` from `#2a2018` to `#3d2e1f` (used in hero only, not footer)
- Add `--cafe-footer-bg: #e8ddd0` (warm light coffee)
- Update `.site-footer` background to `var(--cafe-footer-bg)`
- Update footer text colors: headings → `var(--cafe-gold)`, body text → `var(--cafe-text-muted)`, links → `var(--cafe-text-muted)` with hover `var(--cafe-gold)`
- Bottom bar border → `rgba(45, 33, 23, 0.15)`

### 2. Search Bar Placeholder (`css/main.css`)
**Problem**: In RTL, the search icon sits at `right-3` and placeholder text overlaps it because `padding-right` is only `1rem`.
**Fix**: Increase `padding-right` to `2.75rem` (swap left/right padding values since RTL).

Change `.search-input`:
```css
padding: 0.6rem 2.75rem 0.6rem 1rem;  /* was: 0.6rem 1rem 0.6rem 2.75rem */
```

### 3. About Section Logo (`index.html` line 300)
**Problem**: Uses `no-background-logo-1.webp` — should use `logo-1.webp` (with background) for main display.
**Fix**: Change `src="logo/no-background-logo-1.webp"` → `src="logo/logo-1.webp"`, increase size, remove `opacity:0.6`.

### 4. Footer Credit (`index.html` line 389-391)
**Problem**: Credit text says "طراحی شده با ☕ و عشق".
**Fix**: Replace with clickable link:
```html
<a href="https://amirlwf.ir" target="_blank" class="footer-link" style="color:var(--cafe-gold)">طراحی شده توسط amirlwf</a>
```

### 5. Admin Dark/Light Mode (`css/admin.css` + `admin.html`)
**Problem**: CSS uses `.dark-mode` class but `admin.html` body has no class binding for dark mode. The `x-data="adminPanel"` has `darkMode` state but it's not applied to the body.
**Fix**: 
- In `admin.html` line 31: Add `:class="{ 'dark-mode': darkMode }"` to `<body>`
- In `admin.html` line 58: Add `:class="{ 'dark-mode': darkMode }"` to the wrapper div too (since body class may not cascade into the wrapper)
