# Supabase Integration Plan — کافه آی‌چای

## Overview
Replace localStorage-only data storage with Supabase so the cafe owner can manage the menu from admin.html and changes appear instantly on the public menu page. Uses Supabase Auth for admin protection and Supabase Storage for image uploads.

---

## Step 1: Supabase Project Setup (Manual by User)

User creates a free Supabase project at [supabase.com](https://supabase.com), then runs the following SQL in the SQL Editor:

```sql
-- Categories table
create table public.categories (
  id text primary key,
  name_fa text not null,
  icon text default '',
  "order" int not null default 0,
  created_at timestamptz default now()
);

-- Products table
create table public.products (
  id text primary key,
  category_id text references public.categories(id) on delete set null,
  name_fa text not null,
  description_fa text default '',
  price int not null default 0,
  image_url text default '',
  is_featured boolean default false,
  "order" int not null default 0,
  created_at timestamptz default now()
);

-- Cafe info (single row)
create table public.cafe_info (
  id text primary key default 'singleton',
  name text not null default '',
  tagline text default '',
  phone text default '',
  address_fa text default '',
  instagram text default '',
  telegram text default '',
  hours_fa text default '',
  about_fa text default '',
  welcome_fa text default '',
  created_at timestamptz default now()
);

-- Indexes
create index idx_products_category on public.products(category_id);
create index idx_products_order on public.products("order");
create index idx_categories_order on public.categories("order");
```

**RLS Policies** — public read, authenticated write:

```sql
alter table public.categories enable row level security;
alter table public.products enable row level security;
alter table public.cafe_info enable row level security;

-- Anyone can read the menu
create policy "public_read" on public.categories for select using (true);
create policy "public_read" on public.products for select using (true);
create policy "public_read" on public.cafe_info for select using (true);

-- Only authenticated (admin) can write
create policy "admin_write" on public.categories for all using (auth.role() = 'authenticated');
create policy "admin_write" on public.products for all using (auth.role() = 'authenticated');
create policy "admin_write" on public.cafe_info for all using (auth.role() = 'authenticated');
```

**Storage bucket** — create `cafe-images` (Public = YES) via Dashboard > Storage:

```sql
create policy "public_read" on storage.objects for select using (bucket_id = 'cafe-images');
create policy "admin_upload" on storage.objects for insert with check (bucket_id = 'cafe-images' and auth.role() = 'authenticated');
create policy "admin_delete" on storage.objects for delete using (bucket_id = 'cafe-images' and auth.role() = 'authenticated');
```

**Create admin user**: Dashboard > Authentication > Users > Add User (email + password).

## Step 3: Create `js/supabase-client.js`

Shared data layer wrapping all Supabase CRUD with automatic localStorage fallback:

- `SupaDB.init()` — creates client, returns false if not configured
- `SupaDB.fetchCategories()` / `saveCategory()` / `deleteCategory()`
- `SupaDB.fetchProducts()` / `saveProduct()` / `deleteProduct()`
- `SupaDB.fetchCafeInfo()` / `saveCafeInfo()`
- `SupaDB.uploadImage(file)` — uploads to `cafe-images` bucket, returns public URL
- Every fetch: Supabase → cache to localStorage → return. On error: localStorage fallback → DEFAULT_* fallback.

## Step 5: Modify `js/admin.js`

Major changes:
- **Auth state**: `isAuthenticated`, `loginEmail`, `loginPassword`, `login()`, `logout()`
- **Make all CRUD async**: `saveProduct()`, `saveCategory()`, `executeDelete()`, `saveContent()` all use `await SupaDB.*`
- **Image upload**: `imageMode` (url/upload), `uploading` flag, `handleImageUpload(event)` method
- **Init**: check existing session, load data only if authenticated
