## Task 1: Admin Feedbacks Page

### `js/supabase-client.js` — Add methods (after line 200)

```js
async fetchFeedbacks() {
  if (!this.ready) return Utils.getStorage("cafe_feedbacks", []);
  try {
    const { data, error } = await this.client
      .from("feedbacks").select("*").order("created_at", { ascending: false });
    if (error) throw error;
    return data;
  } catch (e) {
    console.warn("Supabase fetch feedbacks failed:", e);
    return Utils.getStorage("cafe_feedbacks", []);
  }
},

async deleteFeedback(id) {
  if (!this.ready) return this._localDelete("cafe_feedbacks", id);
  const { error } = await this.client.from("feedbacks").delete().eq("id", id);
  if (error) throw error;
},
```

## Task 3: Android Category Chip Fix

### Root causes identified:
1. `fade-in` class (opacity:0) + IntersectionObserver race — chips stay invisible
2. RTL + `overflow-x: auto` + `justify-center` pushes first chips off-screen
3. Hidden scrollbar gives no scroll affordance
4. Deprecated `-webkit-overflow-scrolling: touch`

### Fixes in `css/main.css`:

**Fix 1** — Remove `justify-center` from chip container (index.html line 236), use `justify-start` instead:
```html
<div class="chip-container fade-in mb-8">
```
In RTL, `justify-start` aligns chips to the right (logical start), keeping "همه" visible.

**Fix 2** — Add scroll affordance gradient to `.chip-container`:
```css
.chip-container {
  position: relative;
  /* existing styles... */
  flex-wrap: nowrap;
  -webkit-overflow-scrolling: auto; /* replace deprecated 'touch' */
}
```

**Fix 3** — Fix `fade-in` for chip container specifically: add a rule that `.chip-container.fade-in` gets a shorter transition and is observed immediately, or remove `fade-in` from the chip container entirely (it's below the fold but the observer may not trigger reliably on Android).

**Best approach**: Remove `fade-in` class from the chip container div. The chips should always be visible when the menu section is in view. The section title above it already has `fade-in` for the animation effect.

**Fix 4** — Fix RTL margin on icon span (index.html line 241):
```html
<span x-text="cat.icon" class="me-1"></span>
```
Change `ml-1` to `me-1` (margin-inline-end) for proper RTL spacing.
